How it started

We at Follow Up found that while taking videos, pictures, and simple zoom meetings, a common issue was when you would walk off screen and the video would be stagnant. With lots of thinking, we thought of a solution, what if the camera followed where you went?

Build Start

While developing Follow Up we wanted to create a sleek design that people could easily set up wherever they went. As our product allows for 2 degrees of freedom, we opted to use a core hex motor for one and 2 servos for the other. To mount the servos we used 90-degree servo brackets, attached to a 2-hole Tetrix flat bar. This allowed us to then mount it on a REV extrusion. This was a design choice which would allow the user to adjust the distance between the phone mount to match their own phone.

Build Continued

Next, we mounted the core hex onto a Studica U-channel for stability, which was attached to the REV extrusion with a hex shaft, and a shaft hub. In addition, we 3-D-modelled and printed our phone mounts, which allow the user to set their phone up with a secure fit.

Finished Build

Finally, we mounted a Logitech camera right below the phone mounts, this allows the control hub to receive real-time data about its surroundings and perform further actions given that information. The finished robot comes wired up and can be controlled via wifi allowing for very easy use.

Bugs and Challenges

ERROR

While coding the Follow Up, there was a common issue of the camera not being able to recognize the frame in front of it and perform the correct actions. This is largely part of a low-resolution camera, as well as lighting which greatly impacts the accuracy of the Follow Up. As the code separates the field of view of the camera into 3 separate sections, we can use this information to declare whether the object is on the left, right, or center. These sections are very loosely, defined which created inaccuracies while testing.

Solution

In order, to solve the issue of the camera not being able to detect the spike in contrast relative to the background of the rest of the room, we can isolate what the camera needs to trace with a white background and a single contrasting coloured object in the middle. This is much easier for the camera to pick up which it can then process into tangible information. In order to improve upon the unreliable section boundaries, we tuned the frame width and height until it was able to clearly determine, what was left of the camera, right of the camera, and what was directly infront of it.
